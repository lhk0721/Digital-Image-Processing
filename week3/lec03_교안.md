# lec03.cpp 설명 교안

이 문서는 [`lec03.cpp`](/C:/Users/neighbor/Documents/Code/Github/Digital%20Image%20Processing/Initial%20project/Initial%20project/lec03.cpp) 파일을 **개발 지식이 거의 없는 사람**도 한 줄씩 따라 치며 이해할 수 있도록 풀어쓴 교안입니다.

이번 실습의 목표는 아주 단순합니다.

1. `LENNA.bmp` 파일을 연다.
2. 이미지의 픽셀 값을 메모리로 읽어 온다.
3. 각 픽셀을 `255 - 현재값`으로 바꾼다.
4. 결과를 `output.bmp`로 저장한다.

여기서 중요한 전제는 이 코드가 **그레이스케일(흑백 밝기값) BMP 이미지**를 대상으로 작성되었다는 점입니다.

## 1. 먼저 아주 쉽게 이해할 핵심 개념

### 1-1. 그레이스케일 이미지란?

그레이스케일 이미지는 색깔 정보가 없고, 밝기만 있는 이미지입니다.

- `0`은 검정
- `255`는 흰색
- 그 사이 숫자는 회색

예를 들면:

- `10`은 거의 검정
- `128`은 중간 회색
- `240`은 거의 흰색

이번 실습은 각 숫자를 반대로 뒤집습니다.

- `0 -> 255`
- `255 -> 0`
- `100 -> 155`

그래서 밝은 곳은 어두워지고, 어두운 곳은 밝아집니다.

### 1-2. BMP 파일은 무엇이 들어 있나?

BMP 파일에는 크게 두 종류의 정보가 들어 있습니다.

1. 이미지에 대한 설명서
2. 실제 픽셀 데이터

이 코드에서는 그 설명서를 먼저 읽습니다.

- `BITMAPFILEHEADER`
- `BITMAPINFOHEADER`
- `RGBQUAD` 배열

그 다음에 실제 픽셀 데이터를 읽습니다.

### 1-3. 메모리란?

프로그램은 이미지를 처리하려면 파일 속 숫자를 잠깐 **메모리**라는 작업 공간에 올려야 합니다.

비유하면:

- 파일: 창고에 있는 원본 상자
- 메모리: 책상 위에 꺼내 놓은 작업 재료

코드는 파일에서 이미지를 읽어서 메모리에 올리고, 그 위에서 값을 바꾼 뒤, 다시 파일로 저장합니다.

## 2. 전체 코드 먼저 보기

아래는 원본 코드입니다.

```cpp
#pragma warning(disable:4996)
#include <stdio.h>
#include <stdlib.h>
#include <Windows.h>

int main() {
	BITMAPFILEHEADER hf; //14 byte
	BITMAPINFOHEADER hInfo; // 40 byte
	RGBQUAD hRGB[256]; // 1024 byte

	char path[MAX_PATH];
	GetCurrentDirectoryA(MAX_PATH, path);
	printf("%s\n", path);


	FILE* fp;
	fp = fopen("LENNA.bmp", "rb");
	if (fp == NULL) {
		printf("File not found! \n");
		return-1;
	}
	fread(&hf, sizeof(BITMAPFILEHEADER),1,fp);
	fread(&hInfo, sizeof(BITMAPINFOHEADER), 1, fp);
	fread(hRGB, sizeof(RGBQUAD), 256, fp);
	int ImgSize = hInfo.biWidth * hInfo.biHeight;
	BYTE* Image = (BYTE*)malloc(ImgSize);
	BYTE* Output = (BYTE*)malloc(ImgSize);
	fread(Image, sizeof(BYTE), ImgSize, fp);
	fclose(fp);

	for (int i = 0; i < ImgSize; i++) {
		Output[i] = 255 - Image[i];
	}

	fp = fopen("output.bmp", "wb");
	fwrite(&hf, sizeof(BYTE), sizeof(BITMAPFILEHEADER), fp);
	fwrite(&hInfo, sizeof(BYTE), sizeof(BITMAPINFOHEADER), fp);
	fwrite(hRGB, sizeof(RGBQUAD), 256, fp);
	fwrite(Output, sizeof(BYTE), ImgSize, fp);
	fclose(fp);
	free(Image);
	free(Output);
}
```

### 3-4. BMP 파일의 앞부분을 담을 변수 만들기

```cpp
	BITMAPFILEHEADER hf; //14 byte
	BITMAPINFOHEADER hInfo; // 40 byte

```

이 부분은 BMP 파일의 앞쪽 정보를 담기 위한 변수입니다.

#### `BITMAPFILEHEADER hf;`

- BMP 파일 전체에 대한 기본 정보를 담습니다.
- 예를 들어 "이 파일이 BMP 맞는지", "픽셀 데이터가 어디서 시작하는지" 같은 정보가 들어 있습니다.
- `hf`는 변수 이름입니다.

#### `BITMAPINFOHEADER hInfo;`

- 이미지의 자세한 정보를 담습니다.
- 가로 크기, 세로 크기, 비트 수 같은 정보가 들어 있습니다.

#### `RGBQUAD hRGB[256];`

- 그레이스케일 BMP는 보통 256개의 색상표를 가집니다.
- 사실 그레이스케일에서는 "색상표"라고 해도 검정부터 흰색까지의 단계표라고 생각하면 됩니다.
- `[256]`은 256칸짜리 배열이라는 뜻입니다.

초보자에게 중요한 포인트:

- `hf`, `hInfo`, `hRGB`는 모두 "파일 앞부분 정보를 담는 그릇"입니다.

### 3-5. 현재 작업 폴더 확인

```cpp
	char path[MAX_PATH];

	printf("%s\n", path);
```

이 부분은 이미지 파일을 어디서 찾고 있는지 확인하려고 넣은 코드입니다.

#### `char path[MAX_PATH];`

- `char`는 글자 1개를 저장하는 자료형입니다.
- 하지만 글자 여러 개를 연속으로 모으면 문자열이 됩니다.
- `path[MAX_PATH]`는 경로 문자열을 저장할 공간입니다.

`MAX_PATH`는 윈도우에서 경로 최대 길이로 자주 쓰는 값입니다.

#### `GetCurrentDirectoryA(MAX_PATH, path);`

- 현재 프로그램이 작업 중인 폴더 경로를 읽어 와서
- `path` 변수에 넣습니다.

#### `printf("%s\n", path);`

- 화면에 경로를 출력합니다.
- `%s`는 문자열 출력 형식입니다.
- `\n`은 줄바꿈입니다.

왜 이 코드가 필요할까?

`LENNA.bmp`를 열 때 프로그램은 **현재 작업 폴더**를 기준으로 파일을 찾기 때문입니다.

즉, 파일이 없다고 나오면:

- 코드가 틀린 것이 아니라
- 프로그램이 다른 폴더를 보고 있을 수도 있습니다.

### 3-7. 파일 열기

```cpp
	fp = fopen("LENNA.bmp", "rb");
```

이 줄은 `LENNA.bmp` 파일을 엽니다.

- `fopen`은 파일 열기 함수입니다.
- `"LENNA.bmp"`는 파일 이름입니다.
- `"rb"`는 읽기 전용으로, 바이너리 형태로 연다는 뜻입니다.

여기서:

- `r` = read, 읽기
- `b` = binary, 바이너리

이미지 파일은 글자 파일이 아니라 바이너리 파일이므로 `b`가 필요합니다.

### 3-9. BMP 헤더 읽기

```cpp
	fread(&hf, sizeof(BITMAPFILEHEADER),1,fp);
	fread(&hInfo, sizeof(BITMAPINFOHEADER), 1, fp);
	fread(hRGB, sizeof(RGBQUAD), 256, fp);
```

이 부분은 파일 앞부분의 정보를 읽어 오는 핵심입니다.

`fread`는 파일에서 데이터를 읽는 함수입니다.

형태는 대략 이렇게 이해하면 됩니다.

```cpp
fread(어디에저장할지, 한개크기, 몇개읽을지, 어떤파일에서읽을지);
```

#### 첫 번째 줄

```cpp

```

- `hf` 변수에
- `BITMAPFILEHEADER` 크기만큼
- 1개를
- 파일 `fp`에서 읽어 옵니다.

`&hf`에서 `&`는 "hf의 주소"입니다.

쉽게 말하면:

"읽어 온 데이터를 hf가 있는 메모리 위치에 넣어라"

#### 두 번째 줄

```cpp
fread(&hInfo, sizeof(BITMAPINFOHEADER), 1, fp);
```

위와 같은 방식으로 `hInfo`를 읽습니다.

#### 세 번째 줄

```cpp
fread(hRGB, sizeof(RGBQUAD), 256, fp);
```

- `RGBQUAD` 크기의 데이터 256개를 읽어
- `hRGB` 배열에 저장합니다.

이것이 그레이스케일 색상표입니다.

### 3-10. 이미지 크기 계산

```cpp
	int ImgSize = hInfo.biWidth * hInfo.biHeight;
```

이 줄은 전체 픽셀 수를 계산합니다.

- `hInfo.biWidth` = 이미지 가로 크기
- `hInfo.biHeight` = 이미지 세로 크기

예를 들어:

- 가로 512
- 세로 512

이면 전체 픽셀 수는:

`512 * 512 = 262144`

그레이스케일 8비트 이미지는 픽셀 하나가 1바이트이므로, 이 값은 곧 필요한 메모리 크기와 거의 같습니다.

주의:

실제로 BMP는 줄 맞춤(padding) 문제가 있을 수 있어서 항상 이렇게 단순 계산이 완벽하진 않습니다. 하지만 **이번 실습에서는 기본 개념을 배우는 용도**이므로 이렇게 이해하면 충분합니다.

### 3-11. 원본 이미지용 메모리 만들기

```cpp
	BYTE* Image = (BYTE*)malloc(ImgSize);
```

이 줄은 원본 이미지 데이터를 저장할 메모리를 준비합니다.

#### `BYTE`

- 1바이트 크기의 값입니다.
- 0부터 255까지 저장하기 좋습니다.
- 그레이스케일 밝기값 저장에 딱 맞습니다.

#### `BYTE* Image`

- `Image`는 바이트들이 연속으로 들어 있는 공간의 시작 위치를 가리킵니다.
- 쉽게 말해 "이미지 데이터가 들어 있는 큰 상자의 첫 칸 주소"입니다.

#### `malloc(ImgSize)`

- `ImgSize`만큼 메모리를 빌립니다.
- 실행 중에 필요한 만큼 공간을 만드는 것이므로 **동적 할당**이라고 부릅니다.

왜 배열을 미리 안 만들까?

이미지 크기는 파일마다 다를 수 있기 때문입니다.

예를 들어:

- 256x256 이미지일 수도 있고
- 512x512 이미지일 수도 있고
- 더 클 수도 있습니다.

그래서 실행한 뒤에 실제 크기를 확인하고 필요한 만큼만 메모리를 만드는 것입니다.

#### `(BYTE*)`

`malloc`이 빌려준 메모리를 `BYTE*` 형태로 사용하겠다고 알려 주는 부분입니다.

초보자는 이렇게 이해하면 됩니다.

"빌린 메모리를 바이트 배열처럼 쓰겠다."

### 3-15. 모든 픽셀 반전하기

```cpp
	for (int i = 0; i < ImgSize; i++) {
		Output[i] = 255 - Image[i];
	}
```

이 부분이 이번 실습의 핵심 처리입니다.

#### `for (int i = 0; i < ImgSize; i++)`

`for`문은 같은 작업을 여러 번 반복할 때 씁니다.

이 뜻입니다.

- `i = 0`에서 시작
- `i < ImgSize` 동안 반복
- 한 번 끝날 때마다 `i++`로 1씩 증가

즉, `i`는 0번 픽셀부터 마지막 픽셀까지 차례대로 가리킵니다.

#### `Output[i] = 255 - Image[i];`

현재 픽셀 값을 반전시킵니다.

예를 들어:

- `Image[i] = 0` 이면 `Output[i] = 255`
- `Image[i] = 255` 이면 `Output[i] = 0`
- `Image[i] = 100` 이면 `Output[i] = 155`

왜 `255`일까?

그레이스케일 밝기값의 최대값이 255이기 때문입니다.

이 연산은 "밝기를 뒤집기"라고 생각하면 됩니다.

### 3-17. 헤더 다시 저장하기

```cpp

	fwrite(&hInfo, sizeof(BYTE), sizeof(BITMAPINFOHEADER), fp);
```

출력 파일도 BMP 형식이어야 하므로 앞부분 정보를 먼저 그대로 써 줍니다.

`fwrite`는 파일에 데이터를 쓰는 함수입니다.

형태는 대략 이렇게 이해하면 됩니다.

왜 그대로 쓰나?

이번 실습은 이미지의 밝기값만 바꾸고, 파일 형식 자체는 바꾸지 않기 때문입니다.

### 3-18. 결과 픽셀 저장하기

```cpp
	fwrite(Output, sizeof(BYTE), ImgSize, fp);
```

반전된 픽셀 데이터를 `output.bmp` 파일에 씁니다.

## 5. 초보자가 꼭 헷갈리는 부분

### 5-1. 왜 `Image[i]`처럼 쓰나요?

`Image`는 바이트들이 줄줄이 이어진 메모리 공간의 시작점입니다.

그래서:

- `Image[0]`은 첫 번째 픽셀
- `Image[1]`은 두 번째 픽셀
- `Image[2]`는 세 번째 픽셀

처럼 배열처럼 사용할 수 있습니다.

### 5-2. 왜 `BYTE`를 쓰나요?

그레이스케일 밝기값은 0~255입니다.

이 범위는 1바이트에 딱 들어갑니다. 그래서 `BYTE`가 잘 맞습니다.

### 5-3. 왜 원본과 결과를 나눠 두나요?

한 공간에서 바로 바꿔도 되는 경우가 많지만, 배우는 단계에서는:

- 원본을 보존하고
- 결과를 따로 만들면

생각하기가 훨씬 쉽습니다.

### 5-4. 왜 BMP 헤더를 읽고 다시 쓰나요?

이미지 파일은 픽셀만 있다고 되는 것이 아닙니다.

가로, 세로, 색상표 같은 정보도 같이 있어야 정상적인 BMP 파일이 됩니다.

그래서 출력할 때도 헤더와 색상표를 다시 써야 합니다.

## 6. 이 코드가 가정하고 있는 점

이 실습 코드는 교육용으로 단순하게 작성되어 있습니다. 그래서 아래 내용을 가정합니다.

1. 입력 파일은 8비트 그레이스케일 BMP이다.
2. `LENNA.bmp`가 현재 작업 폴더에 있다.
3. 이미지 데이터를 `가로 x 세로`로 단순 계산해도 문제가 없는 형태다.

실무나 더 정확한 프로그램에서는 아래도 확인해야 합니다.

1. `biBitCount`가 실제로 8인지
2. 압축된 BMP인지 아닌지
3. 줄 끝 padding이 있는지
4. `malloc`이 실패하지 않았는지

하지만 지금은 **이미지를 파일에서 읽어 와서 메모리에서 처리하고 다시 저장하는 큰 흐름**을 배우는 것이 핵심입니다.
